# Incident Response

1. Detect: alert on stale reports, low confidence, rapid transitions, failed submissions, or unexpected reporter activity.
2. Contain: pause report ingestion and disable compromised reporter authorization.
3. Preserve safety: keep withdrawal/repay paths available where contract policy permits.
4. Investigate: preserve logs, report hashes, transaction receipts, source snapshots, and wallet activity.
5. Recover: rotate keys, validate sources, publish fresh reports, run acceptance scenarios, then unpause.
6. Communicate: document affected windows, decisions, and residual limitations.
